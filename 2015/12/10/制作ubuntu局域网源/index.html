一、修改镜像配置文件/etc/apt/mirror.list

Ubunt官方软件仓库在全世界有很多镜像，要想下得快点，当然选国内的源了，并加入一些官方仓库中没有的软件源。由于整个仓库很大，大概在90G左右，用于下载的盘一定要有足够的剩余空间。

二、镜像Ubuntu官方源

打开终端，输入下面的命令，然后就等着吧，好几十个Ｇ呢，镜像完成后会有提示，并返回命令行。

$ sudo apt-mirror

三、拷贝镜像源到局域网服务器上

将下载的源拷贝到局域网服务器的目录下（例如，/data/Ubuntu），由于下载的整个源大约90G左右，该存放源的目录所在分区必须具有足够的空间。

1

$ cp -a /media/ruirui/UbuntuKylin /data/ubuntu/

设置好访问权限，所有用户可读可进入目录

$ chmod a+rx /data/ubuntu -R

四、配置http服务

最常用的自然是Apache了，先安装，再启动http服务。

以Ubuntu 14.04 LTS为例。

$ sudo apt-get install apache2

配置虚拟目录，编辑/etc/apache2/sites-available/default文件，将/data/ubuntu作为web服务的根目录。

默认： DocumentRoot /var/www

修改后： DocumentRoot /data/ubuntu

重新启动web服务 $ sudo apachectl restart

打开浏览器，在地址栏输入：http://服务器IP地址 。如果正确的话，浏览器页面上应该显示Ubuntu目录。 如果局域网内有DNS服务器，可以给该服务器配置一个域名以方便用户端使用。下面以域名ubuntu.beijing.cn为例。

五、用户端安装Ubuntu

从ubuntu官网或ubuntukylin官网下载Ubuntu相应版本的映像，刻盘（建议），安装（过程略）

六、用户端修改源配置文件

修改源配置文件/etc/apt/source.list，以便使用软件源。修改方法如下：

首先备份软件源配置文件

$ sudo cp /etc/apt/sources.list{,.bak}

将其它软件源信息用#注释掉，再将下列软件源信息加入/etc/apt/sources.list中

1
2
3

deb http://ubuntu.beijing.cn/Ubuntu/mirror/mirrors.aliyun.com/ubuntu/ trusty main universe restricted multiverse
deb http://ubuntu.beijing.cn/Ubuntu/mirror/mirrors.aliyun.com/ubuntu/ trusty-security main universe restricted multiverse
deb http://ubuntu.beijing.cn/Ubuntu/mirror/mirrors.aliyun.com/ubuntu/ trusty-updates main universe restricted multiverse

更新本机软件源数据库

$ sudo apt-get update

七、用户端安装软件

现在可以从局域网中安装官方软件仓库中软件包了

$ sudo apt-get install package1 package2 ...

八、更新软件源

如果过一段时间，官方源中的部分软件更新了，则重新执行步骤二将更新的软件包下载到本地存储中（/media/ruirui），即重新运行apt-mirror命令。此时由于只有部分更新，只会下载更新的软件，下载量通常不会很大。

如果隔很长时间才apt-mirror, 那很有可能下载的数据量也会很大的，因为可能很多软件都有更新版本了。

九、将更新的软件同步到局域网服务器上

当然用rsync增量拷贝软件了，不然重新拷贝90多G的数据，会需要很长时间的。用下面的命令：

1

$rsync -auv --delete /media/ruirui/ubuntu /data/ubuntu/
