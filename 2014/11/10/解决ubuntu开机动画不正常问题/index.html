解决Ubuntu开机动画不正常问题

2014-11-10
Linux Questions
约 523 字 预计阅读 2 分钟

文章目录

      • 一、更改/etc/default/grub文件配置
      • 二、更改/etc/grub.d/00_header文件配置

联想的笔记本，显卡NVIDIA GT218M，默认使用开源的驱动，但挂起后，再唤醒就黑屏回不到桌面。经过摸索，解决办法为安装NVIDIA专有驱动：

$sudo apt-get install nvidia-current

解决了唤醒黑屏问题，又带来新的问题：开机logo和开/关机动画均不正常，只显示文字logo。经过摸索，解决办法如下：

一、更改/etc/default/grub文件配置

$sudo gedit /etc/default/grub

GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

改为

GRUB_CMDLINE_LINUX_DEFAULT="video=vesafb vga=845 quiet splash"

其中，845是显示模式号，代表的是1366x768x16M，可根据自己的情况修改为相应的数字。此数据必须正确，而且数字不正确，很可能导致开机画面不能显示或变形。可用下面的命令行来查询显卡支持的模式：

$sudo hwinfo --framebuffer

...

Mode 0x034d: 1366x768 (+5472), 24 bits

#GRUB_GFXMODE=640×480

GRUB_GFXMODE=1366×768

...

上面的0x34d换算成十进制即是845。将640×480的注释去掉，并改为适合自己显示器的分辨率，如1366×768

二、更改/etc/grub.d/00_header文件配置

这一步似乎可以不要

$sudo gedit /etc/grub.d/00_header

set gfxmode=${GRUB_GFXMODE}

在其下一行添加

set gfxpayload=keep

三、更新grub

$sudo update-grub

重启后，应可以看到高清的Grub菜单和开/关机动画 。

以上方法在ubuntu 12.04和ubuntu kylin 14.04测试通过。
