一、简介

Org是一个用文本方式来快速高效地做笔记、维持待办事项和做项目计划的模式。它是一个创作发布系统。 这个文档是Org-mode手册的一个压缩版本,包含了所有的基本功能和命令,以及一些重要的定制提示。 本文档写给不想阅读200多页手册的新手。

1.1 安装

如果你用的是Emacs或者XEmacs包里的Org版本,请跳过本节,直接阅读1.3 节。如果你是从网站上下载的Org版本,无论是zip包还是tar文件或者是Git文件,最好在分发包目录里直接来设置它。把lisp子目录加到Emacs的加载路径里,可以把下面两句加在.emacs文件里:

1
2

(setq load-path (cons "~/path/to/orgdir/lisp" load-path))
(setq load-path (cons "~/path/to/orgdir/contrib/lisp" load-path))

为了提高速度可以用下面的shell命令将Lisp文件编译一下:

make

再把下面一行加到.emacs文件里。它可以使文件中的函数自动加载,而不是启动 Org 模式时立即加载。

1

(require 'org-install)

二、文档结构

Org是基于Outline模式的，它提供了更灵活的编辑结构文件的命令。

2.1大纲

Org是在大纲模式之上实现的。大纲模式可以让我们用层次结构来组织文档，这（至少对我来说）是笔记和想法的最好实现方式。这种结构可以折叠（隐藏）文档的一部分而只显示文档的大概结构,或者只显示我们正在处理的部分。 Org大大简化了大纲模式的使用，它把大纲模式的整个显示/隐藏功能整合到了一个命令中：org-cycle，这个命令绑定到了TAB键上。

2.2标题

标题定义了大纲树的结构。它以处于一行左边缘的一个或多个星号开头。例如：

1
2
3
4
5
6
7
8

* Top level headline
** Second level
*** 3rd level
some text
*** 3rd level
more text

* Another top level headline

如果你不喜欢太多的星号，可以以空格后加一个星号作为标题的开头。查看15.2节[Clean view]，那里有设置方法。

2.3视图循环

大纲模式可以隐藏缓冲区里的部分正文。Org用绑定到TAB和S-TAB上的两个命令来改变视图。

快捷键 功能
TAB 子树循环：当加上一个前缀参数时（C-u TAB），在下面的状态中改变当前子树的视图
FOLDED -> CHILDREN -> SUBTREE
当加上shift键时会触发全局的视图循环
S-TAB、C-u TAB 全局循环：使整个缓冲区在下列状态中循环
OVERVIEW->CONTENTS->SHOWALL
C-u C-u C-u TAB 显示全部，包括drawers

当Emacs刚打开文件时，全局的状态是OVERVIEW，也即只有顶层的标题可见。这可以通过变量org-startup-folded来设置。也可以通过startup关键字设置只对单个文件有效：

1

#+STARTUP: content

5.2 自定义定义不同的TODO状态

可在.emacs配置文件中用TODO关键字定义不同的状态，全局有效

竖直线将TODO关键字（还需要进一步的动作）和DONE状态（不需要进一步的动作）分隔开

例子：

(setq org-todo-keywords

‘((sequence “TODO(t)” “|” “DONE(d)")

(sequence “REPORT(r)” “BUG(b)” “KNOWNCAUSE(k)” “|” " FIXED(f)")

(sequence “|” “CANCELED(c)")))

也可在一个org文件中定义只在文件中有效的TODO状态

例子：

#+TODO:TODO(t) | DONE(d) #+TODO:REPORT(r) BUG(b) KNOWCAUSE(k) | FIXED(f) #+TODO:| CANCELED(c)

当改变这些行中的一行后，光标停留在改变行上，用C-c C-c让改变生效

5.3 进度日志

对于完成的项目，最基本的日志功能是跟踪一个特定项目的完成。Emacs的配置文件中可加入下面的配置：

(setq org-log-done ‘time)

当你将一个项目从一个TODO（未完成）状态改变为一个完成状态时，标题下面就会插入一行“CLOSED:[timestamp]”

(setq org-log-done ‘note)

当你将一个项目从一个TODO（未完成）状态改变为一个完成状态时，会提示你输入一个记录（note），并将它保存在标题为“Closing Note”项目之下。

跟踪TODO状态变换

对于状态记录,Org 可以实现基于每个状态关键字的设置。实现方法是在每个状态关键字后的括号中指定“!”(记录时间戳)或“@”(作一个记录), 这个语法适用于org-todo-keywords。

例如：下面的设置实现，当一个项目设为 DONE 时,会记录时间戳,当状态变为 WAIT 或 CANCELED 时,会作一个记录。

#+TODO:TODO(t) WAIT(W@/!) | DONE(d!) CANCELED(c@)

如果议程文件中的标题带有标签或者带有属性，就可以基于这些元数据筛选标题到议程缓冲区中。

|———+——————————————————————-+——+—| | 快捷键 | 功能 | | | |———+——————————————————————-+——+—| | C-c a m | 将包含指定标签的标题生成一个列表。 | | | | | 筛选规则可以是标签的逻辑表达式，如: +work+urgent-withboss 或 work | home | | | | 如果你经常使用某个搜索，可以将它定义成一个命令。 | | | | C-c a M | 同上，但只选择同时也是TODO项的标题。 | | | |———+——————————————————————-+——+—|

匹配语法

搜索字符串可以使用‘&’作与运算，‘|’作或运算。‘&’的约束力比‘|’的强。

用以搜索的元素可以是标签、匹配标签的正则表达式、或者像PROPERTY OPERATOR VALUE这样带有比较操作符的用来比较属性值的表达式。

例子：

+work-boss : 选择标有:work:，但不标有:boss:的标题

work | laptop : 选择标有:work:或则💻的标题

work | laptop+night : 和前面相同，但要求标有💻的行也要标有:night:

10.3.4 单文件时间轴

时间轴用时间排序视图概述单个文件中的所有带有时间戳的条目。

这个命令的目的是用来给出一个工程中事件的鸟瞰图。

这个议程视图用来对议程文件中的条目进行普通的文本查找。对于查找笔记很有用。

注意：除了议程文件，这条命令也会搜索org-agenda-text-search-extra-files中列出的文件。

查找字符串“+computer +wifi -ethernet-{8.11[bg]}”：搜索包含关键字computer和wifi但不含ethernet，并且不被正则表达式8.11[bg]（排除8.11b和8.11g）匹配的条目。

10.5 定制议程视图

自定义搜索的主要用途是对于频繁使用的搜索进行快捷键绑定，从而快捷地创建议程缓冲区或者稀疏树（当然后者只涵盖当前缓冲区的内容）。

自定义的命令是用变量org-agenda-custom-commands来配置的。你可以用C-c a C来定制这个变量。也可以直接在”.emacs“中用Emacs lisp来设置。

下面的例子包含了所有合法的搜索类型

(setq org-agenda-custom-commands

‘((“w” todo “WAITING”)

(“u” tags “+boss-urgent”)

(“v” tags-todo “+boss-urgent”)))

|—————–+——————————————————————–| | 快捷键 | 功能 | |—————–+——————————————————————–| | C-c a w | 对于包含关键字”WAITING“的TODO项的全局搜索。 | | C-c a u | 对于带有标签”:boss:“而不含标签”:urgent:“的标题的全局标签搜索。 | | C-c a v | 同搜索C-c a u，但搜索范围只限于同时也是TODO项的标题。 | |—————–+——————————————————————–|

十一、富格式导出

11.1 结构的组成元素

文档标题

#+TITLE: This is the title of this Org test file

标题和章节

默认情况下，只有前三个级别用作标题，更深的级别会被看做项目列表。

可以通过变量org-export-headline-levels在全局设置这个开关，或者只是在单个文件中设置：

#+OPTIONS: H:4

目录表

目录表通常会直接插入在文档第一个标题之前。

#+OPTIONS: toc:2 (目录只显示二级标题) #+OPTIONS: toc:nil (无目录)

段落、分行和引用

段落之间至少要有一空行。如果你想实现段内分行，可以在行后加上“\”。

要想在一个区域内实现分行，而其他地方使用正常格式，你可以使用下面的构造，它也可以用来实现诗歌的格式：

#+BEGIN_VERSE Great clouds overhead Tiny black birds rise and fall Snow covers Emacs –AexSchroeder #+END_VERSE

当从另外一个文档中引用一段话时，通常会让它左右都缩进。在Org文档中可以这样作引用：

#+BEGIN_QUOTE Everything should be made as simple as possible, but not any impleer –Albert Einstein

#+END_QUOTE

让文本居中

#+BEGIN_CENTER Everything should be made as simple as possible,\
but not any simpleer –Albert Einstein #+END_CENTER

强调和等宽

粗体

粗体

斜体

/斜体/

下划线

文字两端各加一下划线_

下划线

逐字

=逐字=

代码

~code~

注释行

以‘#‘位于第0列的行会被看作注释，不会被导出。

如果你想要一个缩进的行也被作为注释，用“#+”开头。

以关键字“COMMENT”开头的子树整个树都不会被导出。

被“#+BEGIN_COMMENT“…”#+END_COMMENT”包围的整个区域也都不会被导出。

|—————+————————————–| | 快捷键 | 功能 | |—————+————————————–| | C-c ; | 在一个项的开头增加/删除COMMENT关键字 | |—————+————————————–|

11.2 图片和表格

可以在表格前面用下面几行为表格指定标题和标签，以方便交叉引用。

在文本中可以用\ref{tab:basic-data}来引用它：

#+CAPTION: 表格标题1 #+LABEL: tab:basic-data |——+——| | COL1 | COL2 | |——+——| | 1 | 1 | | 2 | 3 | |——+——|

#+CAPTION: 图片标题1 #+LABEL: fig:leaf [[file:pic/leaf.jpg][leaf-pic]]

this is a leaf picture\ref{fig:leaf}

11.3 纯文本例子

可以包含进来一些纯文本的例子，这些例子会等宽排版，所以适用于代码以及其他类似的情况。

#+BEGIN_EXAMPLE Some example from a text file. #+END_EXAMPLE

为了简单化，一些小型的例子可以冒号开头。冒号前面可以有空格：

Here is an example : Some example from a text file.

对于源代码以及一些其他的文本，可以被Emacs的字体锁（font-lock）特殊标记，也可以让它们像在Emacs的缓冲区中那样显示。

如果要在支持这种语言的专门的缓冲区中编辑例子，将光标置于例子代码中，按快捷键C-c ‘，启动和退出编辑缓冲区。

#+SRCNAME: emacs_lisp_code #+BEGIN_SRC emacs-lisp (+ 1 2 3 4 6 7) #+END_SRC

#+RESULTS:
23

11.4包含文件

当导出文档时，可以包含其他文件中的内容。

导入文件，第二、三个为可选参数，是组织方式：“quote”，“example”,“src”。如果是“src”，语言用来格式化包含的内容。

组织方式是可选的，如果不给出，文本会被当作Org模式正常处理。

十二、导出

Org模式文档可以导出成多种格式：ASCII用于包含在邮件中；HTML用来发布到网页上；LaTeX/PDF用来打印出漂亮的文档；DocBook通过DocBook工具转换成其他各种各样的格式。也可以导出成iCalendar格式，将计划信息并入到桌面日历中

12.1导出选项

导出器能识别缓冲区中提供附加信息的特殊行，这样行可以放在文件中的任何地方。

整个集合可以用C-c C-e #插入到缓冲区中。

#+OPTIONS: ‘:nil *:t -:t ::t <:t H:3 \n:nil ^:t arch:headline #+OPTIONS: author:t c:nil creator:nil d:(not “LOGBOOK”) date:t e:t #+OPTIONS: email:nil f:t inline:t num:t p:nil pri:nil prop:nil stat:t #+OPTIONS: tags:t tasks:t tex:t timestamp:t title:t toc:t todo:t |:t #+TITLE: test #+DATE: <2018-12-15 六> #+AUTHOR: chenbin #+EMAIL: cb@cb-K43SV #+LANGUAGE: en #+SELECT_TAGS: export #+EXCLUDE_TAGS: noexport #+CREATOR: Emacs 24.5.1 (Org mode 8.3.3)

12.2 导出调度器

所有的导出命令都可以通过导出调度器来使用，调度器是一个前缀快捷键，它会提示输入一个字母来指定命令。

通常整个文件都会被导出，但是如果选中区域包含大纲树，就会导出大纲树，并以第一个标题作为文件标题。

|———+————————–| | 快捷键 | 功能 | |———+————————–| | C-c C-e | 用来导出和发布的调度器。 | |———+————————–|

12.3 ASCII/Latin-1/UTF-8的导出

ASCII导出功能能给Org文件提供的一个简单易读的版本，它只包含纯ASCII文本。

Latin-1和UTF-8导出用它们能编码的特殊字符扩展了文件的功能。
