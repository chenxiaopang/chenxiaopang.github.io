一、镜像Ubuntu官方源

利用ubuntu的镜像工具apt-mirror，将官方源镜像到本机或大容量移动硬盘上。例如，/media/cb/CHENBIN/ubuntukylin/2204下。

具体方法见制作Ubuntu局域网源

二、配置本机源

修改/etc/apt/sources.lst，将源指向本机源所所在位置（例如，/media/cb/CHENBIN/ubuntukylin/2204）

三、更新本地软件索引

$ sudo apt-get update

四、更新本地所有软件

$ sudo apt-get dist-upgrade -y

五、修复apt-mirror错误

由于apt-mirror在2017年就停止了维护更新，之后出现的ubuntu发行版软件仓库源更新了仓库的索引文件，apt-mirror没有办法兼容。要想完整地镜像软件仓库中的文件，有两种方法。

1、修改apt-mirror脚本

在apt-mirror脚本中注释掉$uri =~ s/^([^@]+)?@?// if $uri =~ /@/;

sub sanitise_uri {

my $uri = shift; $uri =~ s[^(\w+)://][];

#$uri =~ s/^([^@]+)?@?// if $uri =~ /@/;

$uri =~ s&:\d+/&/&; # and port information

$uri =~ s/~/\%7E/g if get_variable("_tilde");

return $uri;

}

在脚本的下面代码中插入add_url_to_download( $url . $_ . "/cnf/Commands-" . $arch . ".xz" );

add_url_to_download( $url . $_ . "/i18n/Index" );

保存脚本，重启apt-mirror。

2、安装新版本的apt-mirror

1）下载安装修复版apt-mirror

网上有帖子说0.5.4-2版本的apt-mirror包修复了这个bug，重新安装这个包再重新执行apt-mirror命令就可以了。

1
2
3

$ wget https://mirrors.aliyun.com/ubuntu/pool/universe/a/apt-mirror/apt-mirror_0.5.4-2_all.deb

$ sudo dpkg -i apt-mirror_0.5.4-2_all.deb

2）Commands.amd64和Translation-en文件缺失

但是我下载安装之后，还是有问题，cnf文件夹下下载的是Commands.amd64.xz压缩文件，apt-get update却报告需要非压缩文件，我只好挨个去解压一遍。 Commands.amd64文件有了，又出现了Translation-en文件缺失的问题，而i18n文件下是压缩文件Translation-en.gz，同样的问题，只好挨个解压。

问题解决！

3）解压脚本

下面是解压获取Commands.amd64和Translation-en文件的脚本，配合0.5.4-2版本的apt-mirror一起使用。

#!/bin/bash

#Note: to fix apt-mirror problems,after run apt-mirror, run this script to decompress some .xz files

START_PATH=/media/cb/CHENBIN/ubuntukylin/2204/mirror/mirrors.tuna.tsinghua.edu.cn/ubuntu/dists

`find $START_PATH -name "Commands-i386.xz" -exec xz -kdf '{}' ';'`
ret1=$?
   
`find $START_PATH -name "Translation-en.xz" -exec xz -kdf '{}' ';'`
ret2=$?
   
if [ $ret1 -ne 0 -o $ret2 -ne 0 ]
then
    echo "Something error!"
    exit
fi

echo "Success !"
