Hugo是由Go语言实现的静态网站生成器，简单、易用、高效、易扩展、快速部署，能够将Markdown等文本转换为网站静态页面。github pages提供基于代码仓库（repository）的个人页面服务。Hugo和github pages结合起来，可以用来创建个人博客。

一、安装hugo

从ubuntu源里直接安装，或从github上下载最新版。

$ sudo apt-get install hugo

或

1
2
3

wget https://github.com/gohugoio/hugo/releases/download/v0.82.0/hugo_extended_0.82.0_Linux-64bit.deb`

$ sudo dpkg -i hugo_extended_0.82.0_Linux-64bit.deb

二、在github上创建个人代码仓库

https://github.com/chenxiaopang/chenxiaopang.github.io

三、创建个人本地博客

执行下面的命令会在当前目录下生成myblog目录。

$ hugo new site myblog

$ ls myblog

下载的主题会存储在theme目录下。

$ vim config.toml

theme = "even"

五、新建博文

这个命令会在 content 目录下建立 post 目录，并在 post 下生成 Markdown简明教程.md 文件，文章书写就在这个文件里，使用 Markdown 语法完成。

$ hugo new post/Markdown简明教程.md

在文章的front matter 里面，使用draft指定文章是草稿还是可以发表的，draft选项默认为 true，需要改为 false 才能发表文章，建议直接更改archetypes 目录下的 default 文件，把 draft: true 改为 draft: false，这样生成的文章就是默认可以发表的。使用 tags 指定文章的 tag，使用categories 指定文章的类别，示例如下：

1
2
3
4
5

title: "利用Hugo和github pages创建个人博客"
date: 2021-04-17T17:18:26+08:00
draft: true
categories: [Applications]
tags: [blog, github, hugo]

六、本地查看博客效果

下面的命令会将用户的博文生成静态页面，并存放在public目录下

$ hugo server

可在指定的的端口启动hugo服务。

$ hugo server --port=1313

如果要将hugo服务运行在web服务默认的80端口，则需要root权限

1
2

$cd /home/cb/doc/myblog
$sudo hugo server --port=80 &

可以将启动命令加入/etc/rc.local脚本中，开机启动hugo服务

1
2

cd /home/cb/doc/myblog
hugo server --port=80

或者(ssh免密推送)

$ git add .

$ git commit -m "hugo first commit"

$ git push -f upstream master	# 第一次 push 必须使用 -f，因为此时远程仓库非空，和本地有冲突

然后在浏览器中输入地址 http://chenxiaopang.github.io 即可查看部署到远程github上的博客效果。

九、注意事项

1、主页地址格式

必须是“github用户名.github.io”格式，特别是github用户名不能改，如果改成其它名称，页面无法访问。

2、文章时间

front matter中的文章时间不能是未来时间，否则不会生成静态页面。

3、About菜单

在config.toml中增加下面的配置

1
2
3
4
5

[[menu.main]]
  name = "About"
  weight = 50
  identifier = "about"
  url = "/about/"

$ hugo new about.md

会生成content/about.md文件，根据需要进行编辑即可

$ vim content/about.md

4、插入图片

使用markdown语法的话，图片路径使用以static目录为根的相对路径，例如，如果插入图片static/pic/about/lele1.jpg，写法如下：

![](/pic/about/lele1.jpg)

上面的写法，会导致在markdown编辑器中无法显示图片，例如，typora等。对于typora可以”设置图片根目录“或在front matter中设置如下：

typora-root-url: ../static

这样，既能在markdown编辑器中显示插入的图片，上传到github pages上也可以正常显示。

5、大小写问题

Hugo默认会把url中的大写字符转为小写，如果不想转换，在配置文件中加入下面的配置

disablePathToLower = true

Hugo 默认会把分类（categories）名称转为小写，如果不想转换，在配置中加入下面的配置

preserveTaxonomyNames = true
