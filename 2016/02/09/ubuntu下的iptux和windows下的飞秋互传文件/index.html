Ubuntu下iptux和Windows飞秋互传文件

2016-02-09
Linux Application
约 358 字 预计阅读 1 分钟

文章目录

      • 一、问题
      • 二、解决方法
      • 三、启动后图标隐藏不显示

一、问题

当Linux下的iptux是安装源里的版本时，存在下面的问题：

  1. Windows下的飞秋可以向Linux下的iptux发送文件，iptux接受正常
  2. Linux下的iputx给Windows下的飞秋发送目录，飞秋接受正常
  3. Linux下的iputx给Windows下的飞秋发送文件，飞秋接受不正常，接收的文件总是0B

二、解决方法

卸载源里的版本，下载iptux源代码，重新编译安装。

$ sudo apt-get purge iptux
$ sudo apt-get install git libgtk2.0-dev libgconf2-dev g++ make autoconf libtool automake

$ git clone git://github.com/iptux-src/iptux.git
或 
$ cd ~/software/iptux-src; tar xjvf iptux.tar.bz2

$ cd iptux
$ ./configure
或 
$ ./configure --prefix=$HOME/opt/iptux --bindir=$HOME/opt/bin

$make

$ sudo make install //安装到缺省的/usr/local目录下
或
$ make install //安装到Home/opt目录下，不需要sudo

$ iptux

三、启动后图标隐藏不显示

1
2
3

$ killall iptux
$ rm -rf .gconf/apps/iptux/%gconf.xml
$ gconftool-2 --recursive-unset /apps/iptux //将iptux配置信息还原为默认配置

重新启动iptux即可。
